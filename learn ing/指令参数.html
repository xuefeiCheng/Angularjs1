指令的属性：

  • name
  • priority:优先级（多个指令在同一元素执行的优先级）
  • terminal:true/false 如果为true,同一个元素上的其他指令的优先级高于本指令，其他指令将停止执行
  • scope:隔离的scope，此外提供三种方式用于和隔离之外的scope交互 @ & =
  • controller
  • require
  • restrict:限制指令类型
  • template:与templateUrl不共存
  • templateUrl
  • replace:是否用模板替换当前元素。true : 将指令标签替换成temple中定义的内容,页面上不会再有 my-directive 标签；false ：则append（追加）在当前元素上，即模板的内容包在 my-directive 标签内部。默认false。
  • transclude:是否使用ng-transculde来包含html中指令包含的原有的内容，接收两个参数true/false
  • compile
  • link

内部参数：restrict；priority；Template；templateUrl；replace

对外参数：scope；require

行为参数：link；controller

angular.module('app', [])
			.directive('myDirective', function() {
			    return {
			    restrict: String,                
			    priority: Number,
			    terminal: Boolean,
			    template: String or Template Function:
			    function(tElement, tAttrs) {...},
			    templateUrl: String,
			    replace: Boolean or String,
			    scope: Boolean or Object,
			    transclude: Boolean,
			    controller: String or
			    function(scope, element, attrs, transclude, otherInjectables) { ... },
			    controllerAs: String,
			    require: String,
			    link: function(scope, iElement, iAttrs) { ... },
			    compile: // 返回一个对象或连接函数，如下所示：
			    function(tElement, tAttrs, transclude) {
			        return {
			            pre: function(scope, iElement, iAttrs, controller) { ... },
			            post: function(scope, iElement, iAttrs, controller) { ... }
			           }
			        return function postLink(...) { ... }
			        }
			    };
			 });
